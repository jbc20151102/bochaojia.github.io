Projects

Hybrid Movie Recommendation System Optimization March 2026
Problem Pure collaborative filtering (ItemCF) suffers from cold-start issues for new movies, while pure content-based recommendations lack diversity. Business problem: How to optimally blend both approaches to maximize recommendation precision and recall?
Data MovieLens 100K public dataset: 100,000 ratings (1-5 scale) from 943 users on 1,682 movies, plus movie metadata (release year, 19 genre categories).
Approach 1) Built user-item rating matrix and item-based collaborative filtering (ItemCF) similarity; 2) Constructed content feature vectors based on genres and release year; 3) Designed a linear weighted hybrid model: Hybrid Score = α × ItemCF + (1-α) × Content; 4) Used grid search to tune α with F1-Score as the optimization target.
Outcome Grid search identified optimal α = 0.95, achieving the highest F1-Score. Generated 5 types of high-resolution visualizations (rating distributions, model comparison, α-tuning curves). Business value: Delivered a balanced recommendation strategy that can be deployed as a "You May Also Like" module to improve user engagement.
Contribution Individual project: Independently completed data preprocessing, feature engineering, hybrid algorithm implementation, grid search tuning, and all visualization outputs.

Weather Prediction using XGBoost (Group Project) March 2026 – April 2026
Problem Accurate weather prediction is critical for industries like agriculture, retail, and supply chain management. Business problem: Can we build a high-performance predictive model that outperforms traditional forecasting methods?
Data Historical meteorological data including temperature, humidity, wind speed, atmospheric pressure, and precipitation records across multiple timeframes.
Approach 1) Collaborated on data cleaning and feature engineering (handling missing values, creating lag features); 2) My primary contribution: Built and optimized the XGBoost regression model using time series cross-validation to prevent data leakage; 3) Performed hyperparameter tuning via GridSearchCV (optimizing n_estimators, max_depth, learning_rate); 4) Conducted feature importance analysis to identify key meteorological drivers; 5) Integrated XGBoost predictions with other team members' models for ensemble comparison.
Outcome The XGBoost model I developed achieved the lowest RMSE among all individual models tested by the team. Feature importance analysis revealed that humidity and historical pressure trends were the strongest predictors. Business value: Demonstrated that gradient boosting provides a highly accurate, interpretable baseline for operational weather-sensitive business planning.
Contribution Core team member. Independently responsible for the XGBoost pipeline: model selection, time series cross-validation setup, hyperparameter tuning, feature importance analysis, and performance evaluation.
